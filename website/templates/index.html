This project implements a cloud-based real-time control system for multiple vehicles to realize autonomous driving in a controlled environment. The inspiration for this project is from an MIT course, entitled "Robotics: Science and Systems".

The MIT’s project built a self-driving vehicle from a toy race car integrated with an on-board computer, which utilizes a camera and Lidar to perform real-time autonomous driving in a building hallway. All the computing and operation was processed with this on-board computer. However, their project focused on applying only one on-board computer to perform one autonomous driving vehicle without the capability to expand to multiple vehicles. From their implementation, this project will enable the potential to expand the self-driving capability to multiple vehicles by leveraging multi-vehicle management systems to a server. Each vehicle can be set up with the system and access the server to achieve mobile terminal management. Each vehicle can perform environmental detection, information collection, data transmission and collaborative working.

Similar to the MIT project, this project starts with a single agent vehicle controlled by a JETSON TX2 embedded AI computer. With an on-board LIDAR and stereo camera, information gathered from the surrounding environment will be sent to the computer and cloud for processing. In the initial part of the project, the computation would be completed by the on-board computer, and later the computing part will be migrated to the cloud for multiple-agent processing.

Car

This is a self-driving car with powerful control board and multiple sensors.

Board

Jetson TX2 is the fastest, most power-efficient embedded AI computing device. This 7.5-watt supercomputer on a module brings true AI computing at the edge. It's built around an NVIDIA Pascal™-family GPU and loaded with 8GB of memory and 59.7GB/s of memory bandwidth. It features a variety of standard hardware interfaces that make it easy to integrate it into a wide range of products and form factors.

For more details, click here.

Power

To provide sufficient power to JETSON board, we choose POWEROAK 50000mah Portable Battery Pack.

For more details, click here.

Storage

To store the data from multiple sensors, at least a large capacity of 500GB SSD is needed.

For more details, click here.

Controller

Teensy microcontroller recevies control signal from JETSON TX2, then drive the rally cars' servo and motor.

For more details, click here.

Sensors

Three sensors are used: ZED stereo camera, YDLIDAR G4 laser scanner, and IMU.

1. ZED stereo camera, which is a 3D camera, captures environment images, which is used for object detection and object recognition.

2. YDLIDAR G4 laser scanner, which is a 360° lidar, detects the distance to surrounding obstacles, which is used for localization and obstacle avoidance.

3. IMU is used for better localization.

CUDA

JETSON TX2 board uses 256-core NVIDIA Pascal™ GPU architecture with 256 NVIDIA CUDA cores.

For more details, click here.

ROS

To better control the self-driving car, the basic module is developed under ROS Melodic, which is available for Ubuntu 18.04 LTS.

For more details, click here.

Frame

The self-driving car is composed of three layers: Base layer, Main Control layer, and Sensor layer .

1. Base layer: This is the basic of the car, including the chassis, wheels, and motors.

2. Main Control layer: This is the JETSON TX2 control board, which connect with multiple sensors, storage, and power.

3. Sensor layer: This includes all necessary sensors, including stereo camera, lidar, and IMU.

Every two layers are connected by copper pillars. The Main Control layer and the Sensor layer are placed on the self-designed plate.

Camera

1. Download ZED SDK and install on JETSON TX2 board.

2. Open terminal, run the following command to test if the camera works well. 

          /usr/local/zed/tools/ZED Explorer

3. Connect the camera to Jetson board through USB 3.0 port.

For more info about ROS, click here.

Lidar

1. Download ydlidar ROS package into JETSON TX2 board.

2. Clone this package under the ROS folder /catkin/src .

3. Running the following command under /catkin directory to build ydlidar_node and ydlidar_client package. 

          $catkin_make

4. Create the name catkin/dev/ydlidar for YDLIDAR.

5. Open terminal, run the following command to move to startup directory. 

          $roscd ydlidar/startup

6. Run following command to make all files be executable. 

          $sudo chmod 777 ./*

7. Run following command to initialize the environment. 

          $sudo sh initenv.sh

8. If everything is good, then use the following command to test lidar scan result in rviz. 

          $roslaunch ydlidar lidar_view.launch
